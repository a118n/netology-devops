Задача 1
Выберете подходящую систему управления виртуализацией для предложенного сценария. Детально опишите ваш выбор.
Сценарии:
100 виртуальных машин на базе Linux и Windows, общие задачи, нет особых требований

KVM/Xen/vSphere/Hyper-V. Любой из них в принипе подойдет. Если нет лицензий (и желания\возможности тратить деньги), то KVM\Xen.


Преимущественно Windows based инфраструктура, требуется реализация программных балансировщиков нагрузки, репликации данных и автоматизированного механизма создания резервных копий

Hyper-V и vSphere оба подойдут, но делаем выбор в пользу первого по двум причинам: Теоретически Windows-виртуалки будут быстрее на Hyper-V из-за паравиртуализации, и не нужны доп. лицензии (предполагаем, что лицензии на Windows Server уже куплены).


Требуется наиболее производительное бесплатное opensource решение для виртуализации небольшой (20 серверов) инфраструктуры Linux и Windows виртуальных машин

Выбираем KVM, т.к. поддержка Windows в Xen хуже.


Необходимо бесплатное, максимально совместимое и производительное решение для виртуализации Windows инфраструктуры

Если есть лицензия на Windows Server, то Hyper-V. Иначе можно использовать VMware ESXi, либо KVM.


Необходимо рабочее окружение для тестирование программного продукта на нескольких дистрибутивах Linux

KVM/Hyper-V/VirtualBox. Зависит от ОС, на которой производится тестирование. Для Linux\Win 10 Pro используем уже встроенный в систему гипервизор. Для других версий\редакций Windows используем VirtualBox. Можно еще сверху добавить Vagrant для автоматизации разворачивания машин.



Задача 2
Опишите сценарий миграции с VMware vSphere на Hyper-V для Linux и Windows виртуальных машин. Детально опишите необходимые шаги для использования всех преимуществ Hyper-V для Windows.

1. На машину с Windows, с которой есть доступ к хранилищу с виртуальными машинами устанавливаем Microsoft Virtual Machine Converter / StarWind V2V Converter / Paragon V2V Converter или др.
2. Выключаем машину которую собираемся переносить
3. Конвертируем ее с помощью конвертера
4. Импортируем машину в Hyper-V
5. Включаем все галки на вкладке Integration Services.
6. Для поддерживаемых дистрибутивов Linux устанавливаем пакет с Hyper-V Integration Services
7. Удаляем VMware Tools/openvm-tools с машины, при их наличии


Задача 3
Опишите возможные проблемы и недостатки гетерогенной среды виртуализации (использования нескольких систем управления виртуализацией одновременно) и что необходимо сделать для минимизации этих рисков и проблем. Если бы у вас был бы выбор, то создавали ли вы бы гетерогенную среду или нет? Мотивируйте ваш ответ примерами.

Категорически нет, т.к.
1. Гомогенной средой проще управлять
2. Проще мониторить
3. Финансовые расходы на лицензии и поддержку ПО будут меньше
4. Гетерогенная среда - это дополнительная трата времени и ресурсов для приведения нескольких виртуальных сред к единой конфигурации, а так же шаблонов виртуальных машин, политик и пр.
5. Обучение инженеров администрированию нескольких систем опять же приведет к увеличению финансовых расходов, времени и ресурсов
6. В гомогенной среде будет проще организовать Load Balancing, High Availability, Fault Tolerance и Disaster Recovery, не говоря уже о продвинутых фишках вроде DRS, Host-Affinity и тд.
